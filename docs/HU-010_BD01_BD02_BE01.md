## Paso 2 — BD-01 + BD-02: Crear/actualizar `apps/equipos/models.py`

Definir la clase `Equipo` con exactamente estos campos:

| Campo | Definición Django |
|---|---|
| `id` | AutoField PK (implícito) |
| `proyecto` | `ForeignKey('proyectos.Proyecto', on_delete=models.CASCADE, related_name='equipos')` |
| `nombre` | `CharField(max_length=100)` |
| `descripcion` | `TextField(blank=True, default='')` |
| `cupo_maximo` | `PositiveIntegerField()` |
| `fecha_creacion` | `DateTimeField(auto_now_add=True)` |
| `estado` | `CharField(max_length=10, choices=[('activo','Activo'),('inactivo','Inactivo')], default='activo')` |

**Constraint BD-02** — agregar dentro de `class Meta`:

```python
class Meta:
    constraints = [
        models.UniqueConstraint(
            fields=['nombre', 'proyecto'],
            name='unique_nombre_por_proyecto'
        )
    ]
```

`__str__` debe retornar `f"{self.nombre} ({self.proyecto})"`.

## Paso 3 — BE-01: Crear `apps/equipos/serializers.py`

Clase `EquipoSerializer(serializers.ModelSerializer)` con:

- `Meta.model = Equipo`, `Meta.fields = '__all__'`
- Campo `proyecto` marcado como `read_only=True` (se inyecta desde la vista, no desde el body del request).
- Método `validate_cupo_maximo(self, value)` que:
  1. Hace `Parametro.objects.get(clave='max_estudiantes_por_equipo')` y castea `.valor` a `int`.
  2. Si `value > max_valor` lanza `serializers.ValidationError(f"El cupo máximo no puede superar {max_valor} estudiantes por equipo.")`.
  3. Si `Parametro.DoesNotExist`: lanza `serializers.ValidationError("No se encontró el parámetro max_estudiantes_por_equipo en la configuración del sistema.")` — **no usar fallback silencioso**.
  4. Retorna `value` si la validación pasa.

Importar `Parametro` desde el path real confirmado en el Paso 1.
